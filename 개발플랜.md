## 0. 현재 위치 (스냅샷)

- ✅ **1차 UI 프로토타입** 완성 (in-memory, 저장 안 됨)
  - 연간 대시보드(보드 탭): 연도 책장 + 카드 그리드 + 전체 달성률
  - 상단 가로 책갈피 탭: 보드 / 일기 / 일정 / 설정
  - 카드 상세 모달: 날짜 선택 + 날짜별 기록 추가/목록
  - pillyo 브랜딩, Pretendard, 포인트 4색 토큰, 라인 아이콘 세트
- ⬜ 백엔드(Supabase) 미연동 → 새로고침 시 데이터 초기화
- ⬜ 로그인 없음 / 일기·캘린더·체크리스트 미구현

> **다음 큰 갈림길**: (A) UI를 더 다듬고 화면을 늘린 뒤 한 번에 영속화 / (B) 지금 바로 Supabase 붙여 데이터부터 살린다. → **권장: 카드 체크리스트까지 UI 확정(Phase 1) 후 Phase 2에서 영속화.**

## P1. 핵심 UI 완성 (목업 데이터) 🚧

> 백엔드 없이 화면·인터랙션을 먼저 확정. 상태는 useState/로컬 mock으로.

### P1-1. 카드(프로젝트) CRUD
- [ ] `+ 프로젝트 추가` → 생성 모달(이모지·제목·카테고리·색 선택)
- [ ] 카드 수정/삭제 (상세 모달 내 메뉴)
- [ ] 카드 정렬 순서 보존(sort_order 개념만, DnD는 P4)

### P1-2. 체크항목(도장깨기 칸) + 달성률 하이브리드
- [ ] 카드 상세에 체크리스트 추가/완료/삭제
- [ ] 달성률 = 완료/전체 자동 계산
- [ ] 수동 % 슬라이더 + 자동/수동 토글 (기획서 §5 하이브리드)
- [ ] 연도 전체 달성률 = 카드 평균 재계산 연결

### P1-3. 일기 탭
- [ ] 일기 작성 모달: 날짜 + mood 선택(MoodPicker) + 본문
- [ ] 일기 목록(JournalCard) / 빈 상태(EmptyState)
- [ ] (선택) 카드 연결 드롭다운

### P1-4. 일정/캘린더 탭
- [ ] 월간 캘린더 뷰(CalendarCell) — 상단 일정 아이콘과 연결
- [ ] 날짜 클릭 → 일정 추가(제목 + 카드 연결)
- [ ] 카드의 "날짜별 기록"과 캘린더 통합 표시 검토

### P1-5. 설정 탭(껍데기)
- [ ] 달성률 기본 방식, 테마(라이트 고정) placeholder

### P1-6. 반응형/모바일
- [ ] 좌측 연도 책장 → 모바일 서랍/스와이프
- [ ] 상단 탭·모달 모바일 레이아웃 점검

**완료 기준:** 보드/일기/일정 3개 화면이 목업 데이터로 끝까지 동작(추가·수정·삭제·달성률 반영). 디자인 톤 확정.

## P2. 영속화 — Supabase 연동 ⬜

> 기획서 §5 데이터 모델·RLS 기준. UI 확정 후 진입.

### P2-1. 프로젝트/스키마
- [ ] Supabase 프로젝트 생성, 환경변수(.env.local) — `NEXT_PUBLIC_SUPABASE_URL/ANON_KEY`
- [ ] 테이블 생성: `cards / card_items / journals / events` (+ 후속 stickers)
- [ ] **RLS 전 테이블 적용**: `user_id = auth.uid()`
- [ ] 카드 "날짜별 기록" 정식화: `journals.card_id` 활용 or 전용 노트 테이블 결정

### P2-2. 인증
- [ ] Supabase Auth 연동, 로그인/온보딩 화면
- [ ] **Google** 로그인(가장 간단) 먼저
- [ ] **Kakao** 연동
- [ ] **Naver** 커스텀 OAuth (작업량 큼 — 마지막)

### P2-3. 데이터 연동
- [ ] mock → Supabase 쿼리 교체(읽기/쓰기), 낙관적 업데이트
- [ ] 로딩/에러/빈 상태 처리
- [ ] 연도 공간(year) 필터링

**완료 기준:** 로그인 후 카드·일기·일정이 DB에 저장되고 새로고침/다른 기기에서 유지된다.

## 작업 원칙

- **다 넣되 단계로 쪼갠다** — 한 Phase 완료 기준을 만족하기 전 다음으로 넘어가지 않기.
- **디자인 우선** — 일정 압박 없음, 완성도/디자인 품질이 우선 (기획서 §9).
- **저널서클 톤** 유지 — 모던·심플·차분, 파스텔 포인트.
- 커밋은 기능 단위로 자주. 영속화(P2) 전까지는 데이터 비영구임을 명시.
