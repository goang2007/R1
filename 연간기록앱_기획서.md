# pillyo — 연간 기록 앱 기획서

> **서비스명**: **pillyo** (본인이 쓰려고 만든 개인용 사이트)
>
> **한 줄 컨셉**: "나의 한 해를 카드로 쌓고, 도장깨기처럼 달성률을 보는 개인 기록 서비스"
>
> 노션처럼 자유롭지만 복잡하지 않게. 연말에 보면 "올 한 해 내가 뭘 얼마나 했는지"가 한눈에 보이는 나만의 연간 대시보드.
>
> **방향성**: "내가 하고 싶은 건 다 되는 나만의 사이트" — 혼자 쓰고 디자인 연습이 목적이라 기능 욕심은 환영. 단, 한 번에 다 만들지 않고 **다 넣되 단계로 쪼개서** 진행한다.

작성일: 2026-06-12

---

## 1. 누가 · 왜 쓰나

- **사용자**: 본인 (1인). 단, 로그인 계정 기반으로 여러 기기(PC/모바일)에서 동기화.
- **문제**: 노션은 자유도가 높지만 "한 해 단위로 목표를 쌓고 진행률을 보는" 용도로는 세팅이 번거롭고 무겁다.
- **해결**: 연도 → 카드(목표/프로젝트) → 달성률, 그리고 일기·일정까지 한 공간에. 가볍고 직관적인 "연간 회고 + 목표 트래커".

---

## 2. 핵심 개념 (도메인 모델)

```
사용자(User)
 └── 연도 공간(Year)        예) 2026, 2027
      ├── 카드(Card)         "하고 싶은 프로젝트/목표" = 도장깨기 1칸
      │    └── 체크항목(Item) 카드의 세부 단계 → 달성률(%) 계산 근거
      ├── 일기(Journal)      날짜별 기록, 기분(mood)
      └── 일정(Event)        날짜별 일정 (카드와 연결 가능)
```

- **달성률 계산 방식 (택1, 설정 가능)**
  - (A) 체크항목 기반: `완료 항목 수 / 전체 항목 수`
  - (B) 수동 슬라이더: 사용자가 직접 % 조정
- **연도 전체 달성률** = 그 해 카드들의 평균 달성률 (또는 가중치 옵션)

## 3. 기능 명세

### MVP (1차 출시 = "이것만 되면 내가 쓴다")
- [ ] 로그인 — **소셜 선택: Google / Naver / Kakao** (Apple 제외)
- [ ] **연도 책장 네비게이션** — 왼쪽 세로 탭에 연도가 책등처럼 쌓이고, 선택 시 그 연도 메인이 열림
- [ ] 카드(프로젝트) CRUD: 메인의 **`+ 추가` 버튼**으로 프로젝트 카드 생성 → 카드 클릭 시 상세로 진입
- [ ] 카드 달성률 — **체크항목 완료율로 자동 계산 + 언제든 수동으로 % 직접 수정 가능 (하이브리드)**
- [ ] **연간 대시보드(메인)**: 그 해 모든 프로젝트 카드의 달성률 + 내용이 한눈에, 전체 달성 게이지
- [ ] 일기: 날짜별 작성, 기분(mood) 선택, 본문
- [ ] **일정/캘린더 뷰** (월간), 카드↔일정 연결
- [ ] PWA: 홈화면 추가, 오프라인 캐싱(읽기)

### 2차 (있으면 좋은 것)
- [ ] **스티커 기능** — 스티커 보관함 + 일기/카드/캘린더에 붙이기(위치·크기·회전), **직접 만든(업로드/그리기) 커스텀 스티커** 사용 (아래 §스티커 참고)
- [ ] 카드 드래그 정렬, 보관/완료 처리
- [ ] 이미지 첨부(일기/카드)
- [ ] 태그·검색·필터
- [ ] 알림(목표 리마인드) — 푸시

### 나중에 (확장 여지) — *우선순위 낮음, 메인 안정화 후*
- [ ] **통계/회고 리포트** ("2026년의 나" — 월별 활동량, 카테고리 분포, 감정 패턴)
- [ ] 공유/공개 프로필("내 한 해 보기" 링크)

### 네비게이션 구조 (핵심)
- **왼쪽 세로 "연도 책장"**: 연도가 책등(spine)처럼 위→아래로 쌓임. 새 해가 되면 책 한 권이 추가됨. 연도 탭을 누르면 **그 연도의 메인 대시보드**가 열림. (저널서클/레퍼런스의 좌측 세로 탭 패턴을 "연도"로 변형)
- **연도 안쪽**: 메인(카드 보드) ↔ 일기 ↔ 일정 전환.
- 모바일에서는 좌측 책장을 햄버거/스와이프 서랍으로, 연도 안 화면 전환은 하단 탭으로.

## 5. 데이터 모델 (Supabase / Postgres)

```sql
-- 사용자는 Supabase Auth가 관리 (auth.users)

create table cards (
  id          uuid primary key default gen_random_uuid(),
  user_id     uuid references auth.users not null,
  year        int  not null,                 -- 연도 공간
  title       text not null,
  description text,
  category    text,
  emoji       text,
  color       text,
  progress_mode text default 'items',         -- 'items' | 'manual'
  manual_progress int default 0,              -- manual 모드일 때 0~100
  status      text default 'active',          -- active | done | archived
  sort_order  int  default 0,
  created_at  timestamptz default now()
);

create table card_items (              -- 카드 세부 체크항목 (도장깨기 칸)
  id        uuid primary key default gen_random_uuid(),
  card_id   uuid references cards on delete cascade not null,
  title     text not null,
  done      boolean default false,
  sort_order int default 0
);

create table journals (
  id        uuid primary key default gen_random_uuid(),
  user_id   uuid references auth.users not null,
  date      date not null,
  mood      text,                       -- 기분
  content   text,
  card_id   uuid references cards on delete set null,  -- 선택적 연결
  created_at timestamptz default now()
);

create table events (
  id        uuid primary key default gen_random_uuid(),
  user_id   uuid references auth.users not null,
  date      date not null,
  title     text not null,
  card_id   uuid references cards on delete set null,
  created_at timestamptz default now()
);

-- 스티커 보관함 (내가 만든/올린 스티커)
create table stickers (
  id        uuid primary key default gen_random_uuid(),
  user_id   uuid references auth.users not null,
  name      text,
  image_url text not null,            -- Supabase Storage 경로
  source    text default 'upload',    -- 'upload' | 'draw' | 'builtin'
  created_at timestamptz default now()
);

-- 스티커 배치 (어디에 붙였는지 + 위치/크기/회전)
create table sticker_placements (
  id          uuid primary key default gen_random_uuid(),
  user_id     uuid references auth.users not null,
  sticker_id  uuid references stickers on delete cascade not null,
  target_type text not null,          -- 'journal' | 'card' | 'event'
  target_id   uuid not null,          -- 붙인 대상의 id
  x           real default 0,         -- 상대 좌표(0~1) 권장
  y           real default 0,
  scale       real default 1,
  rotation    real default 0,
  z_index     int default 0,
  created_at  timestamptz default now()
);
```

- **달성률 (하이브리드)**: 기본은 `card_items`의 `done` 비율로 **자동 계산**. 사용자가 수동으로 % 를 만지면 `manual_progress`에 저장하고 `progress_mode='manual'`로 전환(자동 무시). 다시 체크항목 기준으로 돌리는 토글 제공.
- **로그인 (Google / Naver / Kakao, Apple 제외)**:
  - **Google** — Supabase Auth 기본 지원, 가장 간단.
  - **Kakao** — Supabase Auth 기본 제공자 목록에 포함, 카카오 개발자 앱 등록 후 연동.
  - **Naver** — Supabase 기본 제공자 아님 → 커스텀 OAuth 연동 필요(직접 OAuth 플로우 또는 Edge Function 경유). 셋 중 작업량 가장 큼.
- **보안**: 모든 테이블에 RLS(Row Level Security) 적용 → `user_id = auth.uid()`인 행만 접근. 혼자 써도 필수.

---

### 스티커 기능 상세
- **보관함**: 내 스티커 그리드. 추가 방법 = (1) 이미지 업로드(PNG, 투명배경), (2) 앱 내 간단 드로잉으로 직접 그리기, (3) 기본 제공 세트.
- **붙이기**: 일기/카드/캘린더 위에 드래그&드롭. 손가락/마우스로 위치·크기·회전 조절 → `sticker_placements`에 저장. 좌표는 대상 영역 대비 상대값(0~1)으로 저장해 반응형에서도 위치 유지.
- **저장**: 스티커 이미지는 Supabase Storage(비공개 버킷 + 본인만 접근), 메타/배치는 DB.
- **디자인 연습 포인트**: 드래그 인터랙션, 핀치/회전 제스처, 레이어(z-index), 캔버스 드로잉 — UI 연습에 좋은 소재.

## 7. 디자인 방향 ("저널서클" 톤 = 모던 & 심플)

**저널서클(JournalCircle) 참고 포인트** (App Store 확인):
- 주간/월간/**연간 보기** 옵션 → 우리 "연도 책장"과 같은 결
- 기분(mood) 기록 + 감정 패턴 인사이트
- 깔끔하고 편안한(clean & calm) 인터페이스
- 레퍼런스 캡처: 좌측 세로 탭(파스텔 색), 부드러운 라운드, 보라 계열 FAB

**1차 디자인 전략**: 먼저 **저널서클과 거의 동일하게** 따라 만든 뒤, 실제 앱을 보며 수정해 나간다.

### 디자인 시스템

**컬러** — 베이스는 화이트/블랙, 아래 4색은 포인트(책갈피·태그·게이지)에만.
```
/* Base */
--white:  #FFFFFF;   /* 배경 */
--ink:    #1A1A1A;   /* 기본 텍스트(블랙) */

/* Point (참조 팔레트) */
--cream:  #E2D9BC;   /* 부드러운 면, 카드 톤, 선택된 책갈피 바탕 */
--blue:   #80ABD7;   /* 보조 포인트, 진행률 게이지 */
--red:    #B20E19;   /* 메인 포인트: 주요 버튼/활성 책갈피/달성 강조 */
--brown:  #683629;   /* 텍스트 강조, 깊은 포인트, 책장 분위기 */
```
- **왼쪽 책갈피 메뉴**: 연도별로 위 포인트 색을 순환 배정(예: 2026=red, 2027=blue…), 활성 연도는 채도↑/펼침.
- 카테고리/기분 태그도 이 팔레트 안에서.

**타이포** — **Pretendard** (전체 폰트). 큰 제목 + 충분한 행간, 가독성 우선.
```
font-family: "Pretendard", system-ui, sans-serif;
```

**형태/모션**
- 둥근 모서리(rounded-2xl), 부드러운 그림자, 카드 중심 레이아웃. 좌측 연도 책장은 책등 메타포.
- 과하지 않은 페이드/슬라이드, 달성률 게이지 채워지는 애니메이션.
- 다크모드는 추후(1차는 라이트 우선).

- ✅ **프로젝트 목적**: 본인이 쓰려는 용도 + **디자인 연습**. → 개발 기간보다 **완성도/디자인 품질 우선**, 일정 압박 없음.

- ✅ **디자인**: 저널서클 톤 (모던·심플·차분, 파스텔 포인트)
- ✅ **스티커**: 커스텀(업로드/그리기) 스티커를 일기·카드·캘린더에 붙이기. 2차 범위.

## 10. 진행 현황 (2026-06-12 업데이트)

### 확정된 추가 결정
- ✅ **서비스명 = pillyo**: 본인이 필요해서 만드는 개인용 사이트. 워드마크/메타데이터에 적용.
- ✅ **상단 가로 "책갈피" 탭**: 저널서클의 *좌측 세로 섹션 탭*(Todo/Routine/Event/Checklist…)을 **상단 가로 탭**으로 변형. 현재 탭 = **보드 · 일기 · 일정 · 설정**, 각 탭에 아이콘 + 파스텔 포인트 색.
  - ※ **연도 책장(YearSpine)은 그대로 좌측 세로 유지** — 가로화한 건 "연도"가 아니라 "섹션(화면 전환)" 탭이다. (둘은 다른 네비게이션 축)
- ✅ **카드 상세보기 = 날짜별 기록**: 카드 클릭 → 모달에서 **날짜 선택(date picker) + 본문**으로 기록을 추가하고, 날짜 최신순 목록으로 누적. 카드 앞면엔 "기록 N" 개수 노출.
  - 데이터 모델 매핑: 기획서 §5의 `card_items`(체크 단계)와 별개로, 이 "날짜별 기록"은 향후 **카드에 연결된 일기/메모** 성격 → `journals.card_id` 또는 카드 전용 노트 테이블로 정식화 예정.
- ✅ **공용 라인 아이콘 세트** 도입(보드/일기/일정/설정/캘린더/추가/닫기) — currentColor 기반 SVG.
